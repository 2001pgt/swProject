# FootFit - 재료 기반 음식 추천 웹사이트
## 프로젝트 발표자료

---

## 📋 프로젝트 개요

**FootFit**은 사용자가 가지고 있는 재료를 입력하면 만들 수 있는 요리를 추천해주는 웹사이트입니다. 
빠르고 정확한 레시피 검색과 맞춤형 추천 시스템을 통해 사용자의 요리 경험을 향상시킵니다.

### 🎯 주요 목표
- 사용자 친화적인 재료 기반 레시피 검색
- 정확한 매칭 알고리즘을 통한 최적의 요리 추천
- 개인화된 서비스 제공

---

## 🔍 검색 알고리즘

### 핵심 로직

```sql
SELECT
    fr.id,
    fr.food_name,
    fr.recipe,
    COUNT(DISTINCT fi.ingredient_name) as matching_ingredients,
    total_ingredients.total_count as total_ingredients,
    (COUNT(DISTINCT fi.ingredient_name) * 100.0 / total_ingredients.total_count) as match_percentage
FROM food_recipes fr
JOIN food_ingredients fi ON fr.food_name = fi.food_name
WHERE (사용자_입력_재료_매칭_조건)
GROUP BY fr.id, fr.food_name, fr.recipe, total_ingredients.total_count
HAVING match_percentage > 0
ORDER BY match_percentage DESC, matching_ingredients DESC
LIMIT 20
```

### 알고리즘 특징

1. **재료 매칭률 계산**
   - 사용자 입력 재료와 레시피 재료 간의 일치도 계산
   - `매칭_재료_수 / 전체_재료_수 * 100`으로 백분율 계산

2. **기피 재료 필터링**
   - 사용자가 설정한 기피 재료가 포함된 레시피는 제외
   - 개인화된 추천 서비스 제공

3. **정렬 기준**
   - 1차: 매칭률 내림차순 (높은 매칭률 우선)
   - 2차: 매칭 재료 수 내림차순 (더 많은 재료 매칭 우선)

4. **결과 제한**
   - 상위 20개 레시피만 반환하여 성능 최적화

## 📊 API 엔드포인트

### 인증 API
- `POST /api/auth/register` - 회원가입
- `POST /api/auth/login` - 로그인

### 레시피 API
- `GET /api/recipes/search?ingredients=재료1,재료2` - 재료 기반 레시피 검색
- `GET /api/recipes/:id` - 레시피 상세 정보
- `GET /api/recipes/ingredients/list` - 모든 재료 목록

### 재료 API
- `GET /api/ingredients` - 전체 재료 목록
- `GET /api/ingredients/search?q=검색어` - 재료 검색

## 💾 데이터베이스 설계

### 주요 테이블

1. **food_recipes** - 레시피 기본 정보
   - id, food_name, recipe, cooking_time, difficulty

2. **food_ingredients** - 레시피별 재료 정보
   - food_name, ingredient_name, quantity

3. **users** - 사용자 정보
   - id, username, email, password_hash

4. **user_disliked_ingredients** - 사용자 기피 재료
   - user_id, ingredient_name
